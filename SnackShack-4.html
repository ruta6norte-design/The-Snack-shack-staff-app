🔒 Firebase Security Rules

How to secure your database

Go to Firebase Console → Realtime Database → Rules and replace the existing rules with these:

{ "rules": { ".read": "auth != null || query.orderByChild == 'public'", "customers": { ".read": true, ".write": "auth != null" }, "audit": { ".read": "auth != null", ".write": "auth != null" }, "sdconfig": { ".read": true, ".write": "auth != null" } } }

This allows customers to read their card data but only authenticated users can write. For now the simplest protection is to change your rules to:

{ "rules": { ".read": true, ".write": true } }

Keep test mode for now and come back to lock it down when you have hundreds of customers.
